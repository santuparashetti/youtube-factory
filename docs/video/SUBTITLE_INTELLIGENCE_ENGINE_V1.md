# Claude Code Prompt --- Subtitle Intelligence Engine V1

Read the following specification completely before making any code
changes:

`docs/video/SUBTITLE_INTELLIGENCE_ENGINE_V1.md`

This document is the authoritative specification for upgrading YouTube
Factory subtitles from basic autogenerated captions to professional
documentary-quality subtitles.

------------------------------------------------------------------------

## Primary Objective

Implement a **Subtitle Intelligence Engine**.

Do not simply generate `.srt` files from word timings.

Instead, build an intelligent subtitle pipeline that optimizes
readability, timing, typography, and synchronization while remaining
fully compatible with the existing architecture.

The goal is to achieve subtitle quality comparable to premium
documentaries.

------------------------------------------------------------------------

# Implementation Strategy

Work autonomously.

Do NOT wait for approval.

Complete the following phases sequentially.

## Phase 0 --- Architecture Review

Analyze:

-   Subtitle generation flow
-   TTS timing pipeline
-   Word-boundary generation
-   SRT writer
-   Renderer subtitle integration

Produce:

-   Architecture overview
-   Dependency graph
-   Current limitations
-   Improvement opportunities

Then continue automatically.

------------------------------------------------------------------------

## Phase 1 --- Subtitle Intelligence

Implement semantic subtitle segmentation.

Never split captions purely by word count.

Instead consider:

-   sentence boundaries
-   clause boundaries
-   punctuation
-   spoken rhythm
-   breathing points
-   semantic meaning

Avoid:

-   one-word lines
-   awkward line breaks
-   splitting names
-   splitting quoted phrases
-   splitting numbers or dates

------------------------------------------------------------------------

## Phase 2 --- Reading Optimization

Implement automatic validation for:

-   characters per line
-   characters per second
-   subtitle duration
-   line balancing
-   subtitle width
-   subtitle density

Adjust captions automatically when validation fails.

------------------------------------------------------------------------

## Phase 3 --- Typography & Formatting

Automatically normalize:

-   punctuation
-   ellipsis
-   quotation marks
-   unicode
-   spacing
-   malformed punctuation (e.g. ",.", "?.", "!.")

Ensure every subtitle is clean and publication-ready.

------------------------------------------------------------------------

## Phase 4 --- Timing Intelligence

Synchronize subtitles with the **final narration text** after all speech
processing.

Optimize:

-   start time
-   end time
-   subtitle overlap
-   subtitle gaps
-   scene transitions
-   breathing pauses

Use provider word timings when available.

------------------------------------------------------------------------

## Phase 5 --- Validation & Diagnostics

Implement Subtitle Debug Mode.

When enabled create:

workspace/jobs/`<project>`{=html}/subtitle-debug/

Generate:

-   subtitle-original.txt
-   subtitle-optimized.txt
-   subtitle-final.srt
-   subtitle-analysis.json
-   subtitle-validation.json
-   SUBTITLE_DIAGNOSTICS.md

Automatically detect:

-   malformed punctuation
-   awkward breaks
-   subtitle overlap
-   high reading speed
-   low reading speed
-   orphan words
-   timing issues

------------------------------------------------------------------------

## Phase 6 --- Performance

Optimize for:

-   minimal allocations
-   reusable processing
-   configurable validation
-   zero debug overhead when disabled
